## 2. 本地开发最小配置

### 阶段1: 纯开发调试（Mac本地即可）

**硬件要求**:
- **CPU**: M1/M2/M3 Mac（8核以上）
- **内存**: 16GB起步，建议32GB
- **存储**: 100GB可用空间

**能做什么**:
- ✅ 前后端开发（React + FastAPI）
- ✅ 数据处理流程（解析、清洗、分块）
- ✅ pgvector本地测试（<1万文档）
- ✅ Ollama运行7B模型（Qwen2.5-7B）
- ❌ 大规模向量化（内存限制）
- ❌ 生产级性能测试

---

### 阶段2: 完整MVP测试（需GPU服务器）

**硬件要求**:
- **开发机**: 本地Mac（代码开发）
- **GPU服务器**: RTX 4090 (24GB) + 64GB RAM
  - 购买成本: $3K-4K（二手市场）
  - 租用成本: ¥2K-3K/月（云GPU服务）

**能做什么**:
- ✅ 完整RAG流程测试
- ✅ 10万+文档向量化
- ✅ 14B模型推理
- ✅ 性能基准测试
- ✅ 多用户并发测试

## 4. 详细实施计划

### Week 1-2: 本地环境搭建

**目标**: 在Mac本地跑通最小可用系统

```bash
# 1. 安装基础环境
brew install postgresql@15
brew install redis
brew install ollama

# 2. 启动Docker Compose
docker-compose up -d postgres redis

# 3. 安装pgvector扩展
psql -d mydb -c "CREATE EXTENSION vector;"

# 4. 下载并运行7B模型
ollama pull qwen2.5:7b

# 5. 启动Streamlit原型界面
streamlit run app.py
```

**验收标准**:
- [ ] 文档上传功能正常
- [ ] 向量化流程正常（1000份文档 <10分钟）
- [ ] RAG问答响应<10秒
- [ ] 准确率>70%（测试集）

---

### Week 3-4: 云GPU功能扩展

**目标**: 租用云GPU测试完整MVP功能

**推荐云GPU平台**:

| 平台 | 配置 | 价格 | 特点 |
|------|------|------|------|
| **AutoDL** | RTX 4090 24GB | ¥2.5/小时 | 国内访问快，中文界面 |
| **RunPod** | RTX 4090 24GB | $0.34/小时 | 国际稳定，按秒计费 |
| **Lambda Labs** | A100 40GB | $0.80/小时 | 企业级稳定性 |

**部署步骤**:
```bash

# 2. 下载14B模型

# 3. 向量库迁移到Qdrant
# 导出pgvector数据 → 导入Qdrant

# 4. 性能测试
python benchmark.py --docs 100000 --queries 1000
```

**验收标准**:
- [ ] 10万文档向量化完成（<4小时）
- [ ] 查询响应<3秒（P95）
- [ ] 准确率>80%
- [ ] 并发10用户稳定运行

## 9. 关键决策点

### 决策1: 何时从本地迁移到云GPU？

**触发条件**（满足任一即可迁移）:
- Mac本地完成基础功能验证（2-4周）
- 测试数据量>1万份
- 需要测试14B或更大模型
- 需要性能基准测试

---

### 决策2: 何时考虑采购硬件？

**触发条件**（需同时满足）:
- 云GPU累计使用>3个月
- 月度云成本>¥2000
- 项目确定长期投入（>6个月）
- 团队规模>3人

**ROI分析**:
```
RTX 4090成本: $3500
云GPU成本: ¥2500/月 ≈ $350/月

回本周期: $3500 / $350 ≈ 10个月

结论: 使用>10个月建议采购
```

---

### 决策3: 自建 vs 云端？

**选择自建**（需同时满足）:
- 数据敏感度高，必须本地部署
- 用户规模>100
- 有专职运维团队
- 预算充足（>$30K）

**选择云端**（满足任一）:
- 快速验证POC
- 用户规模<100
- 无专职运维
- 需要弹性扩展
