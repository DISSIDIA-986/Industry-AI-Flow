Industry AI Flow 架构楼层图

按 6 层展示组件与依赖。连线已按真实主链路校正：请求流、控制流、数据流、安全/观测流。

L1 体验层 (UI/Client)
L2 接入层 (API/Auth/Cache)
L3 编排层 (Workflow/Intent/Prompt)
L4 AI Runtime (RAG/Dispatch/Exec/Cost)
L5 数据层 (Postgres/Prompt/Artifact)
L6 平台层 (Security/Obs/Config)
请求主路径（实线）
编排控制流（实线）
数据读写流（虚线）
安全/观测流（虚线）
L1 体验层

Streamlit / Web UI

对话、上传、查询、可视化交互入口。

Prompt Admin

模板编辑、实验配置、版本管理。

External API Client

第三方系统通过 REST API 对接。

User Input

自然语言问题、文档、成本估算请求。

L2 接入层

FastAPI Routers

workflow/query、dispatch、prompt、cost-estimation 等路由入口。

Auth + Tenant Scope

认证、租户作用域与权限控制。

Query Cache

多租户缓存入口，减少重复查询开销。

Input Validation

Pydantic 参数校验与请求清洗。

L3 编排层

Workflow Orchestrator

节点编排、执行顺序控制、节点耗时统计。

Intent Classifier

知识检索/代码执行/成本估算等意图识别。

Prompt Selector/Manager

模板选择、渲染、实验策略、版本控制。

Routing & Budget Policy

local_only / hybrid_auto / cloud_only 路由与预算策略。

L4 AI Runtime

RAG Engine

检索、重排、上下文拼接与 grounded 输出。

LLM Dispatch Service

本地优先调度、云回退、调用与成本记录。

Code Execution Manager

Docker / provider fallback 执行分析代码。

Cost Estimation Model

训练、加载、预测与置信区间估计。

L5 数据层

PostgreSQL + pgvector

文档索引、向量检索、业务与审计数据。

Prompt / Usage / Budget Tables

prompt_*、llm_usage_logs、llm_budget_policies 等。

Model Artifacts

workspace/models/cost_estimation/*.json。

Document Chunks

原文、分块、embedding 与 metadata。

L6 平台层

Redaction + Egress Guard

敏感信息脱敏，限制明文出站。

Observability + Audit

workflow/dispatch 指标、审计日志与回放。

Config + Release Gates

配置开关、phase/release gate 与回滚脚本。

Runtime Infrastructure

Docker runtime、provider health、隔离执行环境。

主查询链路：L1 -> L2 -> L3 -> L4 -> L5。
成本估算链路：L1 -> L2 -> L4 -> L5。
安全与观测作为横向能力，覆盖 API、编排、运行时与数据层。
