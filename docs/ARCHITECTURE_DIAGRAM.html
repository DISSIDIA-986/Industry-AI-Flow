B. As-Is vs Target Execution Views

Frontend calls /api/v1/workflow/query as default orchestration path
↓
IntentClassificationWorkflow is primary runner
↓
Agent dispatch returns templated response path
Fallback orchestrator pipeline only on init failure
↓
RAG executes strongly via /api/v1/rag/query
Cost ML executes strongly via /api/v1/cost-estimation/*
Advanced analytics executes via /api/v1/unified/query and /api/v1/data/analyze

Single entry orchestration with explicit node contracts
↓
Intent -> Route -> RAG / Cost ML / Dynamic Analytics in one executable graph
↓
Online confidence gate + optional query rewrite pass
Standardized safety / redaction / egress controls
Unified response + provenance + cost metadata
↓
One trace path, one quality gate, one policy control plane
