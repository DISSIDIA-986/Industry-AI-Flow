## ⚠️ 问题与挑战

### 1. 准确率未达标 ❌

**问题**: 准确率 20%，远低于 70% 目标

**根本原因分析**：

1. **向量检索不精准**（权重40%）
   - 文档块数量少（23块），导致语义匹配不准确
   - 示例：查询"什么是RAG系统"未检索到 `1_rag_system.txt`
   - 向量相似度计算受文档覆盖度限制

2. **关键词匹配过严**（权重30%）
   - 测试脚本要求答案包含ALL期望关键词
   - 示例：答案包含"Retrieval Augmented Generation"但缺少中文"增强"即判错
   - 语义正确但措辞不同也会被判错

3. **文档内容覆盖不足**（权重20%）
   - 6个文档无法覆盖所有20个测试问题
   - 部分问题（如"context window"）文档内容不够详细
   - 缺少专门针对测试问题的术语对照

4. **LLM回答风格**（权重10%）
   - qwen2.5:7b 倾向使用技术术语和长句
   - 回答准确但不包含测试期望的简短关键词

**优化尝试记录**：
- ✅ 优化文档内容，添加中英文对照
- ✅ 调整 top_k: 3→5
- ✅ 优化 prompt 模板
- ✅ 减小 chunk_size: 500→300（块数 13→23）
- ❌ 准确率仍未达标

### 2. 依赖兼容性问题 ⚠️

**已修复问题**：

1. **PyMuPDF 编译失败**
   - 问题: PyMuPDF 1.23.8 需从源码编译，Python 3.13 失败
   - 解决: 使用预编译 wheel（PyMuPDF 1.26.5）
   - 教训: 使用灵活版本号（`>=` 而非 `==`）

2. **Pydantic v2 兼容性**
   - 问题: `BaseSettings` 移至 `pydantic-settings` 包
   - 解决: 安装 `pydantic-settings>=2.0.0`，修改import
   - 教训: 主动适配最新依赖版本

3. **pgvector 向量类型转换**
   - 问题: `operator does not exist: vector <=> numeric[]`
   - 解决: 显式转换 `%s::vector`
   - 教训: 数据库类型需要显式转换
