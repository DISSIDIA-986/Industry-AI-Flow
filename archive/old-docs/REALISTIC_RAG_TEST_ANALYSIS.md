## 8. 系统瓶颈与优化建议

### 8.1 性能瓶颈

#### 响应时间过长 (平均14.5秒)

**分解延迟**:
```
查询embedding: ~200ms
BM25检索: ~20ms
向量检索: ~500ms
Reranker: ~1000ms
LLM生成: ~12秒 ← 主要瓶颈
────────────────────
总计: ~14秒
```

**优化方案**:

1. **缓存机制** (预期节省30%)
   - 查询embedding缓存
   - 相似问题结果缓存
   - Reranker结果缓存

2. **模型优化** (预期节省40%)
   - 使用更快的LLM(GLM-4-Flash vs qwen2.5:7b)
   - 减少生成token数(设置max_tokens=200)
   - 流式输出提升用户体验

3. **并行处理** (预期节省20%)
   - Embedding和BM25并行
   - 多文档并行重排序

**目标**: 从14.5秒降至 **5秒以内**

### 8.2 检索质量瓶颈

#### 关键词匹配率偏低 (34.3%)

**根本原因**:

1. **CSV转文本信息损失**
   - 原始数据: 545行 × 13列 = 7,085个数据点
   - 转换后: 2,174字符 ≈ 保留<1%信息
   - 损失: 具体数值、排序关系、统计细节

2. **文档块粒度问题**
   - 当前: 800字符/块
   - 问题: 一个数据集被切分成4-5块，关联信息分散

3. **缺少领域文档**
   - General RAG问题全部失败
   - 缺少基础概念文档(BM25, embeddings, etc.)

**优化方案**:

1. **改进CSV处理** ⭐⭐⭐⭐⭐
   ```python
   # 当前: 只转换统计摘要
   # 改进: 为每个数据分析问题预生成答案文档

   示例:
   Q: "哪个省份失业率最高?"
   预计算: "1976年1月British Columbia失业率14.2%最高"

   Q: "装修vs未装修价格?"
   预计算: "装修房均价5.2M, 未装修4.8M, 差价400K"
   ```

2. **添加元数据过滤** ⭐⭐⭐⭐
   ```python
   # 检索时按数据集过滤
   if "housing" in question.lower():
       filter = {"dataset": "Housing"}
   ```

3. **扩充基础知识库** ⭐⭐⭐⭐
   - 导入samples/目录的RAG文档
   - 添加数据分析基础概念

## 10. 最佳实践建议

### 10.1 数据导入策略

**For CSV/结构化数据**:

1. **不要只转换统计摘要** ❌
   - 信息损失严重
   - 无法回答具体问题

2. **预计算常见查询** ✅
   - 最大值/最小值/平均值
   - 分组统计
   - 排名和排序
   - 时间趋势

3. **保留多维度视图** ✅
   - 按类别分组的统计
   - 时间序列摘要
   - 地理维度汇总

### 10.2 问题路由策略

```python
if question_type == "knowledge_retrieval":
    # 使用RAG - 适合概念性问题
    use_rag_agent()

elif question_type == "data_analysis":
    # 使用数据分析Agent - 适合计算/统计问题
    use_data_analysis_agent()

elif question_type == "hybrid":
    # 先RAG检索上下文，再调用数据分析Agent
    context = rag_retrieve()
    result = data_analysis_agent(context + question)
```

### 10.3 答案验证

对于数据分析类问题，建议添加验证步骤:
```python
if answer_contains_statistics():
    # 从原始数据验证
    verify_with_pandas(csv_path, answer)
```
